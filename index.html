Installation de JEEDOM sur un Orange Pi +2

Initialisation de l'Orange Pi +2

1. Telecharger l'image Armbian pour OPI+ 2 sur leur site officiel Choisir UBUNTU SERVER - LEFGACY KERNEL

2. Formater la micro SD avec SDformatter puis charger l'image sur la même micro SD dument formatée avec Win32DiskImager

3. Insérer la carte micro SD dans l’OrangePI+ 2 brancher clavier, écran, et l'alimentation. Apres démarrage s’identifier avec les identifiant/mot_de_passe par défaut: root/1234 . Confirmez une nouvelle fois 1234 , puis créez lui un nouveau MOT_DE_PASSE . Confir mez MOT_DE_PASSE . Il vous demande maintenant de creer un nouvel utilisateur. Déclinez en pressant simultanément CTRL+C

4. Maintenant on va copier le contenu de la carte Micro SD sur la mémoire EMMC avec la commande suivante : sudo nand - sata - install (A cceptez tout en appuyant 3x sur ENTER). Cela dure 30 minutes environ. A la fin choisir Power OFF

5. Eteindre l'Orange PI+ 2 sudo poweroff retirer la SD du lecteur, Rebranchez l'alimentation pour démarrer l’OPI+2... Là on va le configurer. Clavier : dpkg - reconfigure keyboard - configuration

Generic 105 key (intl)PC - > French - > Français - > The default for the keyboard layout - > No compose key Timezone : dpkg-reconfigure tzdata Choisissez EUROPE - > PARIS, c'est fait!

6. Initialiser le WIFI par la commande suivante : sudo nmtui Choisissez ACTIVATE A CONNECTION - > Choisissez votre wifi - > Entrez le MdP, c'est fait!

6.5 Redémarrez pour rendre effectifs les changements sudo reboot

7. Ajouter le répertoire de paquets de Ondrej pour le php : sudo add - apt - repository ppa:ondrej/php

8. Actualisez les sources sudo apt - get update puis, redémarrez pour rendre effectifs les changements sudo reboot

9. Appliquez les changements avec les nouvelles sources sudo apt - get upgrade Cette Opération dure environ 25 minutres.

sudo reboot

Préparation et installation du logiciel

10. Installez manuellement MYSQL avant d'executer le script jeedom sudo apt - get install mysql - server (Definir le MDP pour root pendant l’installation : MOT_DE_PASSE_DE_VOTRE_CHOIX)

11. installer depuis une source github , la dernière version de JEEDOM. Code : Tout sélectionner wget https://raw.githubusercontent.com/jeedom/core/stable/install/install.sh

12. Rendre executable install.sh chmod +x install.sh (rend exécutable install.sh)

13. Exécuter install.sh en donnant le mot de passe prédéfini durant l'installation sudo ./install.sh – m MOT_DE_PASSE_DE_VOTRE_CHOIX L’opération va durer environ 20 minutes, JEEDOM va s’installer. Une foi l’installation finie, redémarrer sudo reboot Maintenant, on peut passer sur l’ordianteur pour avoir acces à Jeedom via le navigateur internet à l’adresse ip fournie par le routeur, ou via l’invité de commande BASH dispo sous win10 ou Ubuntu en connexion SSH.

Egalement on peut se connecter en SSH avec FileZilla pour visualiser graphiquement les fichiers du serveur.
14. Une fois installé se connecter en SSH à travers BASH. Par la commande ssh 192.168.0.xx Si le message « Warning remote host identification changed » apparait c’est normal, pas de panique. Cela survient généralement après une reinstallation. C’est parce que la version SSH - Client (qui essaie de se connecter à SSH - Server située sur le serv eur que l’on essaie de joindre) détecte un changement de serveur pour la même adresse IP. Etant donné qu’ on a formaté cela parait logique . Cette sécurité empêche qu’un pirate fasse un clone de ton serveur sans que tu ne t’en re ndes compte. Exécuter le code suivant permet à SSH - Client d’oublier l’ancien serveur. Un foi sa mémoire vide il acceptera ne nouveau. ssh - keygen – R 192.168.0.xx Cette commande signifie : Vider le cache concernant cette IP.

Sécurisation du serveur Linux

15. Maintenant on va empêcher l ’accès ROOT à SSH. Comme ça si jamais ne quelqu’un prend le contrôle d’un compte utilisateur via SSH pourra tout faire sauf nous empêcher de nous connecter à ROOT via un câble HDMI et un Clavier en direct sur le serveur. Cela ne te déconnectera pas, mais e mpêchera seulement les connexions à l’avenir. sudo nano etc/ssh/sshd_config Cherchez « PermitRootLogin yes » et remplacer par « PermitRootLogin no ».
